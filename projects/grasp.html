Project GRASP

Generalizable Robotic Action Models for Sensorimotor Policies

Can robots learn generalizable manipulation skills from a handful of demonstrations?

Overview

GRASP is a research project investigating how robots can learn to manipulate objects from limited human demonstrations and generalize these skills beyond the exact conditions under which they were trained.

Classical robotics requires accurate system models, controlled environments, and deterministic pipelines. These systems fail in unstructured real-world settings where object positions, lighting, backgrounds, and scene clutter vary unpredictably. Robot learning through imitation offers flexibility — but faces a fundamental constraint: data scarcity. Unlike language or vision, there are no internet-scale robot datasets.

This project explores six complementary experimental threads, each probing a different angle on the same core question: How do we train sensorimotor policies that generalize from fewer demonstrations?

Motivation

Human beings generalize manipulation skills effortlessly. We:

  • Build object-centric representations that abstract away irrelevant scene structure
  • Learn invariant features robust to changes in lighting, background, and viewpoint
  • Maintain world models that support planning and prediction under uncertainty

Standard end-to-end pixel-to-action models struggle with generalization because they learn spurious correlations — background colors, camera angles, irrelevant nearby objects — rather than the true causal structure of the manipulation task. We investigate structural inductive biases as the lever: modifications to representation, architecture, perception pipeline, and policy conditioning that encode the right prior knowledge without requiring thousands of demonstrations.

Research Directions

1. ACT Architecture Modifications

Can conditioning the CVAE encoder on visual observations improve policy representation quality?

The standard ACT CVAE encoder processes only joint states and action sequences. Our modification adds ResNet18-encoded image features into the encoder, so the latent code is conditioned on both what the robot sees and where it is.

Result: Modified ACT achieves 27.8% lower validation loss (0.0931 vs 0.1289).

✅ Complete

2. Vision-Only ACT (LeRobot)

Does removing proprioceptive inputs force the policy to learn more generalizable visual representations?

Removed all joint angle inputs from ACT policy. The model receives only image observations — no end-effector positions, no joint angles. Delivered as a HuggingFace-compatible drop-in module for LeRobot.

Why it matters: A policy that relies on joint angles can achieve low training loss by trajectory memorization. Visual grounding is a prerequisite for genuine generalization.

✅ Complete

3. Classical Pipeline

How much can explicit structure — open-vocabulary detection + geometric planning — achieve without any demonstrations?

Uses open-vocabulary visual detector (prompted with natural language) to localize objects, computes 3D coordinates via camera intrinsics + depth, plans top-down grasps geometrically, and executes via IK.

Result: ~90% success in simulation on fixed objects. New objects handled by changing text prompt.

🔄 Ongoing

4. GradCAM for Vision-Language-Action Models

Can we build attribution maps that show which image regions drive each action dimension, conditioned on the language instruction?

The problem: Standard ViT-based VLAs use mean pooling before the action head, giving every patch the same gradient — making GradCAM produce uniform, meaningless saliency maps.

The fix: Replace mean pooling with cross-attention, where the text instruction queries the visual patches.

Result: Cross-instruction saliency correlation reduced from 0.896 → 0.134 (85% improvement).

✅ Complete

5. RoboPrompt — In-Context Learning for Robots

Can LLMs specify complex robot behaviors via in-context learning, with only a handful of demonstration examples in the prompt?

Uses an LLM to convert natural language task descriptions into discretized robot actions via in-context examples. A bridge module converts discretized actions to continuous poses and then to joint commands via IK.

Why ICL? Fine-tuning requires large datasets. ICL requires 3–10 examples, generalizes without gradient updates, and works with any LLM API.

🔄 Ongoing

6. Multi-task ACT

Can a single ACT policy learn to perform multiple distinct manipulation tasks, and how do we mitigate negative task transfer?

Train ACT on combined datasets from 7 MetaWorld tasks. Explored unconditioned shared policy, task-embedding-conditioned policy, and mixture-of-experts architectures.

Result: Unconditioned multi-task policy averaged ~45% success vs ~65% for per-task oracles. Adding task embedding raised this to ~55%.

🔄 Ongoing

Lessons Learned

1. Data Diversity is the Primary Bottleneck
Training ACT with demonstrations from a single fixed initial state yielded 0% evaluation success despite excellent training loss. After re-collecting demonstrations from randomized initial states, evaluation success improved substantially.
2. Architecture Improvements Are Necessary But Not Sufficient
The modified CVAE encoder produced a 27.8% improvement in validation loss, but this architectural gain was masked by the data diversity problem. Both improvements are required together.
3. The Mean-Pooling Trap in VLA Interpretability
Any mean-pooling operation before the prediction head makes gradient attribution meaningless — every patch receives the same gradient weight. Attention-weighted pooling restores spatial gradient information.
4. Classical Pipelines Quantify the Value of Demonstrations
The classical pipeline achieves strong performance on simple tasks without any demonstrations. This provides a direct baseline: if your learned policy cannot exceed 90% on these tasks, something is wrong with your training setup.
5. Multi-task Transfer Depends on Task Similarity
Tasks with similar motor patterns transferred positively (reach ↔ pick-place), while contact-rich tasks with different force profiles interfered (push ↔ door-open). Task conditioning mitigates this.

Future Directions

  • Object-centric representations: Replace dense image features with explicit object-level tokens as policy inputs
  • Foundation model integration: Use pre-trained visual foundation models (DINOv2, SAM) as frozen vision encoders
  • Scaling GradCAM to large VLAs: Extend attention-pooling approach to OpenVLA and RT-2-scale models
  • World model-augmented planning: Integrate learned dynamics model for imagined rollouts
  • Multi-step RoboPrompt tasks: Scale language-conditioned ICL to longer-horizon tasks with sub-goal decomposition
  • Curriculum learning for multi-task ACT: Design curricula that reduce negative transfer using task similarity metrics
  • Real-world evaluation: Deploy full pipeline on real robot arm with varied lighting and object configurations
