Embodied AI & Legged Locomotion

● Active · StochLab, RBCCPS, IISc · Dr. Shishir N. Y. Kolathaya (May 2026–Jul 2026)   iBot Club, IIT Madras (May 2025–Apr 2026)

"How do we train locomotion controllers that transfer to real hardware with minimal gap — and generalize to terrain the simulator never showed?"

At StochLab (RBCCPS, IISc Bangalore), I design reward functions and RL architectures for quadruped locomotion training on NVIDIA Isaac Sim. The research component focuses on the latest RL architectures for locomotion (curriculum learning, privileged teacher-student distillation, contact-implicit planning). The engineering component is closing the sim-to-real gap on physical robots from Strider Robotics — tuning actuator models, domain randomization ranges, and deployment pipelines.

Prior work through Project GRASP at iBot Club investigated generalizable manipulation: vision-language-action (VLA) models with ACT architectures, GradCAM interpretability for VLAs, and in-context learning for robot control via RoboPrompt. Six experimental threads produced concrete results including a 27.8% reduction in validation loss via image-conditioned CVAE encoders and 85% improvement in saliency map discriminability.

Methods

  • Proximal Policy Optimization (PPO)
  • Teacher-student distillation
  • Domain randomization
  • Behavior cloning from demos
  • VLA models (ACT, LeRobot)
  • GradCAM for interpretability

Results

  • Sim-to-real deployment on quadruped
  • 27.8% lower ACT validation loss
  • 85% saliency correlation reduction
  • ~90% sim success w/ classical pipeline
  • HuggingFace-compatible VLA module

LLM Agent Evaluation & Benchmarking

● NeurIPS 2026 · ACL 2026

"How do we measure whether LLM agents can reason about scientific artifacts — not just generate plausible text about them?"

MLAuditBench is the first interactive RL environment built specifically to evaluate LLM agents on ML experiment integrity auditing. The key design insight: static code analysis tools miss the cross-artifact, sequential reasoning that real auditing demands. Agents must cite verbatim evidence, submit verdicts within step budgets, and contend with adversarially clean episodes and planted red herrings. GPT-4.1-mini matches or exceeds trained human auditors at all difficulty tiers. o4-mini systematically hallucinates. No automated agent reaches the interactive expert ceiling of 0.954.

TRELLIS addresses a complementary problem in financial agents: LLMs are good at orchestration but must not be allowed to compute. The architecture enforces bounded orchestration — intent understanding, routing, and synthesis live in the model; all analytical truth comes from deterministic tool pipelines. This prevents execution-time drift between planning and tool outputs, a failure mode previous systems ignored.

MLAuditBench Design

  • 62 synthetic experiments
  • 8 violation types (Kapoor-Narayanan)
  • 5 ML archetypes, 3 difficulty tiers
  • Evidence-citation requirement
  • Red-herring & fabrication penalty
  • OpenEnv-compliant Docker release

Key Results

  • GPT-4.1-mini: 0.900/0.901/0.819
  • Human baseline: 0.82/0.86/0.57
  • Expert ceiling: 0.954
  • o4-mini: systematic hallucination
  • Inference bugs inflate variance

Neuroscience-Inspired Computing

SENAI Lab, IIT Madras (Sep–Nov 2025)   Biotech Club, IIT Madras (May 2025–Apr 2026)

"Can biological learning principles — spike timing, predictive coding, synaptic homeostasis — be made computationally precise enough to implement on hardware?"

Project NeuroSpike explored three layers simultaneously: simulation (Brian2 models of LIF, AdEx, Izhikevich neurons), theory (predictive coding frameworks, differential geometry of neural manifolds), and hardware (LIF neuron circuits in Verilog on FPGA). The aim is adaptive, low-power intelligent systems that bridge computational neuroscience, machine learning, and neuromorphic computing. The team of 4 worked under an EE-23b project lead and a CSE professor at IIT Madras. Work presented at the IIT Madras Research Conclave in November 2025.

At SENAI Lab, the focus was on the mathematical foundations: differential geometry for neural representation, PCA/ICA for latent structure, and critical reading of theoretical neuroscience literature toward novel publishable contributions.

Topics

  • Spiking neural networks (SNNs)
  • STDP learning rules
  • Predictive coding
  • LIF / AdEx / Izhikevich neurons
  • Synaptic homeostasis
  • Continual learning (no forgetting)

Implementation

  • Brian2 Python simulations
  • Verilog FPGA circuits
  • Pygame neural visualizations
  • Differential equation solvers
  • Research Conclave presentation

Neural Signal Decoding & Neuroprosthetics

● Active · CFI, IIT Madras (Apr 2026–present)

"Can we decode user intent from EEG and EMG signals accurately enough to drive an upper-limb prosthetic in real time — without requiring surgical implants?"

Regensis is an AI-driven neuroprosthetic for upper-limb rehabilitation. The system integrates neural signal processing (artifact removal, feature extraction from EEG/EMG), motor synergy analysis, CNN-based intent decoding, and musculoskeletal simulation to enable intuitive prosthetic control. The goal is a non-invasive system that learns individual motor patterns and adapts over rehabilitation sessions.

Technical Pipeline

  • EEG / EMG signal acquisition
  • Artifact removal & preprocessing
  • Motor synergy decomposition
  • CNN intent classification
  • Musculoskeletal simulation
  • Prosthetic actuation interface

Design Goals

  • Non-invasive signal decoding
  • Real-time inference pipeline
  • Per-user adaptive calibration
  • Upper-limb rehabilitation focus

AI for Scientific Computing & Chemical Engineering

● Completed · IIT Madras · Prof. Abhinav S. Raman (Jan 2026–May 2026)

"Can an LLM agent replace a CFD engineer's setup workflow — reading a problem statement and producing a validated simulation with no manual configuration?"

Under the guidance of Prof. Abhinav S. Raman (Dept. Chemical Engineering, IIT Madras), I built an LLM-orchestrated agent that converts natural language problem descriptions into validated OpenFOAM CFD simulations end-to-end. A schema-constrained NL-to-CFD pipeline parses problem descriptions into structured specifications, a Reynolds-aware model selection matrix (12 solvers, 13 turbulence models) picks the right configuration deterministically, and an LLM mesh generation engine produces blockMeshDict/snappyHexMesh configs before executing and validating the run. Achieved 100% pass rate across 5 canonical flow regimes (lid-driven cavity, cylinder Re=100, backward-facing step, pipe flow, channel flow) with sub-6% error on literature reference values. The system operates as a module within a multi-agent scientific platform also integrating DWSIM (process simulation) and hybrid RAG for broader chemical engineering workflow automation.

System Components

  • Multi-agent orchestration layer
  • OpenFOAM CFD integration
  • DWSIM process simulation
  • Hybrid RAG for domain knowledge
  • Async dependency-aware execution

Design Properties

  • Full pipeline traceability
  • LLM natural language interface
  • Fault tolerance for long jobs
  • Simulation result validation
