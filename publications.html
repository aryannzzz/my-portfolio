MLAuditBench: An Interactive Reinforcement Learning Environment for Evaluating LLM Agents on ML Experiment Integrity Auditing

Aryan Jain et al.

Reproducibility failures in published machine learning (ML) research are widespread, yet existing detection tools rely on static code analysis that cannot capture the cross-artifact, sequential reasoning real auditing demands. We introduce MLAuditBench, the first reinforcement learning (RL) environment designed for ML experiment integrity auditing. Agents interact with synthetic experiments spanning 8 violation types from the Kapoor-Narayanan taxonomy, 5 ML archetypes, and 62 unique experiments across three difficulty tiers. Rather than simply flagging violations, agents must cite exact quoted evidence from the artifacts they inspect, and submit a verdict within a fixed step budget. To prevent gaming, the benchmark includes adversarially clean episodes, planted red herrings, and a fabricated-evidence penalty, with reward structured as 80% violation detection, 10% efficiency, and 10% verdict accuracy. We evaluate three frontier OpenAI models, an interactive Claude Sonnet 4.6 expert audit, a Qwen2.5-72B open-weight baseline, and a 10-seed ML-trained human baseline, all under identical grading conditions. GPT-4.1-mini matches or surpasses the human baseline on every tier (0.900/0.901/0.819 vs. 0.82/0.86/0.57), while o4-mini suffers systematic hallucination failures, and all automated agents fall below the interactive expert ceiling of 0.954, confirming the benchmark discriminates at frontier capability levels. MLAuditBench is released as an OpenEnv-compliant Docker container with fixed seeds, full per-seed logs, and a public human-evaluation kit.

When LLMs Orchestrate but Do Not Compute: Tool-Grounded Agents over Structured Streaming Financial Data

Aryan Jain (sole author) · IIT Madras · ch24b040@smail.iitm.ac.in

Large language models are increasingly used as interfaces to analytical systems, but live high-stakes domains require more than fluent generation: outputs must remain grounded, current, and auditable. We present TRELLIS, a streaming-native, tool-grounded agent architecture built around bounded orchestration: the language model handles intent understanding, routing, tool planning, and response synthesis, while deterministic pipelines remain the sole source of analytical truth. Unlike prior tool-use systems that assume a static or implicitly consistent world state, we explicitly address execution-time drift between planning and tool outputs through structured execution semantics. MCTS-guided routing with an LLM-as-Value-Function handles cross-domain tool selection, and a temporal consistency constraint (max|τ(z_i)−τ(z_j)| < ε) prevents execution-time drift across streaming tool outputs. The system preserves structured intermediate artifacts throughout execution and supports interaction across cashflow, bonds, equities, foreign exchange, and market news.
